NAMA PROJECT
============
TRACE (Tracking Report & Activity Control Engine)
Aplikasi laporan harian lapangan berbasis mobile web untuk pencatatan aktivitas 
pekerjaan, progres, kondisi lapangan, dokumentasi foto, dan monitoring laporan.


DESKRIPSI PROJECT
=================
Project ini adalah aplikasi web berbasis CodeIgniter 4 yang digunakan untuk:
- Input laporan harian lapangan
- Penyimpanan draft laporan
- Review dan submit laporan final
- Monitoring laporan berdasarkan user, tanggal, dan status
- Rekap untuk admin dan manager
- Export laporan ke PDF
- Integrasi pengiriman ringkasan laporan ke WhatsApp

Framework: CodeIgniter 4
Database: MySQL
Language: PHP 8.2+
Frontend: HTML5, CSS3, JavaScript, PWA (Progressive Web App)

SETUP DI LOCAL
==============

1. REQUIREMENT SYSTEM
   - PHP 8.2 atau lebih tinggi
   - MySQL/MariaDB 5.7+
   - Composer (untuk package management)
   - Git (untuk version control)
   - Text editor/IDE (VS Code, PhpStorm, dll)

2. LANGKAH SETUP AWAL
   a) Clone/copy project ke folder lokal
   b) Buka terminal di folder project
   c) Install dependencies:
      $ composer install

3. KONFIGURASI .ENV (File Environment)
   a) Copy file 'env' menjadi '.env'
      $ cp env .env
   
   b) Edit file .env dan sesuaikan konfigurasi:
      CI_ENVIRONMENT = development
      
      app.baseURL = 'http://localhost:8080/'
      
      database.default.hostname = localhost
      database.default.database = bytecorner_db
      database.default.username = root
      database.default.password = (kosongkan atau sesuaikan)
      database.default.DBDriver = MySQLi
      database.default.port = 3306

4. SETUP DATABASE
   a) Buat database MySQL baru:
      CREATE DATABASE bytecorner_db;
   
   b) Import schema dari file:
      $ mysql -u root -p bytecorner_db < db/bytecorner_db_schema.sql
   
   c) Alternatif import data lengkap (dengan master data):
      $ mysql -u root -p bytecorner_db < db/bytecorner_db_all_data.sql

5. JALANKAN APLIKASI
   Gunakan development server CodeIgniter:
   $ php spark serve
   
   Default server akan jalan di: http://localhost:8080/


BASH COMMANDS YANG SERING DIGUNAKAN
===================================

1. DEVELOPMENT SERVER
   $ php spark serve                    (Run dev server on http://localhost:8080)
   $ php spark serve --port 3000        (Custom port, contoh 3000)

2. DATABASE MIGRATION
   $ php spark migrate                  (Run all pending migrations)
   $ php spark migrate:rollback         (Rollback last batch)
   $ php spark migrate:refresh          (Rollback all + run migrate)
   $ php spark migrate:status           (Show migration status)

3. SEED DATABASE
   $ php spark db:seed Database\\Seeder (Run seeder class)

4. GENERATE MODELS/CONTROLLERS
   $ php spark make:model MyModel       (Generate model class)
   $ php spark make:controller MyController  (Generate controller)
   $ php spark make:migration create_table_name  (Generate migration)

5. CACHE & DEBUG
   $ php spark cache:clear             (Clear application cache)
   $ php spark db:seed                 (Seed database)

6. TESTING
   $ composer test                     (Run PHPUnit tests)
   $ vendor/bin/phpunit               (Alternative way to run tests)

7. COMPOSER
   $ composer install                  (Install all dependencies)
   $ composer update                   (Update dependencies)
   $ composer dump-autoload            (Refresh autoloader)


ROUTES & PAGES MAPPING
======================
Project ini menggunakan role-based access control dengan 3 role utama:
1. Supervisor/PIC (Pelaksana) - User biasa, input laporan
2. Manager - Monitoring laporan tim
3. Admin - Kelola user dan master data

ROUTE STRUKTUR:
- Routes tanpa filter: Accessible to anyone (guest)
- Routes dengan 'auth': Require login untuk semua role
- Routes dengan 'role:' filter: Restricted by role

3. REPORT PAGES - USER UMUM (Dengan Filter 'auth')
   ===============================================

   A. CREATE REPORT PAGE
      URL: /reports/create (GET)
      Method: DailyReportController::create
      Model: DailyReportModel, ReportPhotoModel, ReportHeavyEquipmentUsageModel, ReportWorkerUpdateModel
      Service: DailyReportService
      View: app/Views/Reports/CreateReportPage.php
      Deskripsi: Form input laporan harian baru (bertahap per section)
      Flow:
        - Halaman form dengan berbagai section:
          * Basic Info (tanggal, lokasi, cuaca)
          * Equipment (alat berat yang digunakan)
          * Worker Update (update kondisi pekerja)
          * Notes & Photos (catatan dan dokumentasi)
        - Form options diambil dari: HeavyEquipmentCategoryModel, WorkerCategoryModel
        - User bisa simpan draft atau submit

   B. EDIT REPORT PAGE
      URL: /reports/edit/{id} (GET)
      Method: DailyReportController::edit
      Model: DailyReportModel, ReportPhotoModel, ReportHeavyEquipmentUsageModel, ReportWorkerUpdateModel
      Service: DailyReportService, AuthService
      View: app/Views/Reports/CreateReportPage.php
      Deskripsi: Edit draft laporan yang sudah tersimpan
      Flow:
        - Load existing report data via DailyReportService::getReportBundle()
        - Check permission: user hanya bisa edit laporan miliknya
        - Load existing photos dan equipment data
        - Return form dengan data yang sudah fill
      Validasi: User hanya bisa edit laporan dengan status 'Draft'

   C. SAVE DRAFT REPORT
      URL: /reports/save-draft (POST)
      Method: DailyReportController::saveDraft
      Model: DailyReportModel, ReportPhotoModel, ReportHeavyEquipmentUsageModel, ReportWorkerUpdateModel
      Service: DailyReportService
      View: -
      Deskripsi: Save laporan ke database sebagai DRAFT
      Flow:
        - Validate input
        - Insert/update DailyReportModel dengan status 'Draft'
        - Insert photos ke ReportPhotoModel
        - Insert equipment to ReportHeavyEquipmentUsageModel
        - Handle file upload untuk dokumentasi

   D. REVIEW REPORT PAGE
      URL: /reports/review/{id} (GET)
      Method: DailyReportController::review
      Model: DailyReportModel, ReportPhotoModel, ReportHeavyEquipmentUsageModel, ReportWorkerUpdateModel
      Service: DailyReportService, ReportSummaryService, AuthService
      View: app/Views/Reports/ReviewReportPage.php
      Deskripsi: Review checklist sebelum final submit
      Flow:
        - Load report bundle (report + related data)
        - Generate preview ringkasan untuk WhatsApp
        - Show all data dalam readonly mode
        - Button: Approve untuk submit atau back untuk edit

   E. SUBMIT REPORT (FINAL)
      URL: /reports/submit/{id} (POST)
      Method: DailyReportController::submit
      Model: DailyReportModel, AuditLogModel
      Service: DailyReportService, ReportSummaryService
      View: -
      Deskripsi: Final submit laporan (change status dari Draft ke Submitted)
      Flow:
        - Validate report is still Draft
        - Check permission
        - Update DailyReportModel::status = 'Submitted'
        - Create audit log entry
        - Optional: Send WhatsApp summary via Notification::sendToWhatsApp()

   F. DETAIL REPORT PAGE
      URL: /reports/detail/{id} (GET)
      Method: DailyReportController::detail
      Model: DailyReportModel, ReportPhotoModel, ReportHeavyEquipmentUsageModel
      Service: DailyReportService
      View: app/Views/Reports/DetailReportPage.php
      Deskripsi: Lihat detail laporan (readonly view)
      Flow:
        - Load complete report bundle
        - Show all data in formatted display
        - Show photos dengan preview
        - Show edit/delete options jika user adalah owner dan status masih Draft

   G. EXPORT REPORT TO PDF
      URL: /reports/pdf/{id} (GET)
      Method: DailyReportController::pdf
      Model: DailyReportModel, ReportPhotoModel, ReportHeavyEquipmentUsageModel
      Service: DailyReportService, ReportPdfService
      View: app/Views/Reports/PdfTemplate.php
      Deskripsi: Generate dan download laporan dalam format PDF
      Flow:
        - Load report data
        - Render HTML template (PdfTemplate.php)
        - Convert to PDF menggunakan DOMPDF library
        - Force download file PDF

DEVELOPMENT WORKFLOW
====================

1. BUAT FITUR BARU:
   a) Buat route di app/Config/Routes.php
   b) Buat controller method di app/Controllers/
   c) Buat model jika perlu database query
   d) Buat service untuk business logic
   e) Buat view file di app/Views/
   f) Test di browser atau API testing tool

2. HANDLE DATABASE CHANGE:
   a) Create migration: php spark make:migration add_column_to_table
   b) Edit migration file di app/Database/Migrations/
   c) Run migration: php spark migrate
   d) Update model properties sesuai kebutuhan

3. ADD NEW DEPENDENCY:
   a) Install via composer: composer require vendor/package
   b) Commit composer.lock
   c) Update production .env jika ada konfigurasi baru

4. DEBUG:
   - Check app/writable/logs/ untuk error logs
   - Use dd() function untuk dump & die
   - Enable debug toolbar via CI_ENVIRONMENT = development
   - Check browser console untuk JS errors


PENTING UNTUK PRODUCTION
=========================

1. File .env HARUS dikonfigurasi:
   - CI_ENVIRONMENT = production
   - app.baseURL = actual domain
   - app.forceGlobalSecureRequests = true (HTTPS)
   - database credentials sesuai production DB

2. Folder writable/ MUST WRITABLE:
   chmod -R 755 writable/
   chmod -R 755 public/Uploads/

3. Set proper file permissions:
   chmod 644 app/Config/*.php
   chmod 755 app/

4. Enable SSL/HTTPS untuk PWA functionality

5. Setup backup strategy untuk database

6. Monitor logs di writable/logs/ secara berkala

CATATAN TAMBAHAN
================

1. ROLE PERMISSION:
   - Supervisor/PIC: Bisa create/edit laporan sendiri, view own reports
   - Manager: Bisa lihat laporan tim, monitoring, trend analysis
   - Admin: Full access, manage users, monitoring semua laporan

2. REPORT STATUS WORKFLOW:
   Draft → (submit) → Submitted → (review) → Reviewed
   
   Hanya user dengan role Manager/Admin yang bisa "review" laporan.

3. SECURITY:
   - Password di hash dengan bcrypt (default CodeIgniter)
   - Session menggunakan CodeIgniter session handler
   - JWT token untuk API dengan Firebase JWT library
   - SQL injection protection via ORM dan parameter binding
   - CSRF protection via CodeIgniter token

4. ENVIRONMENT VARIABLES PENTING:
   - CI_ENVIRONMENT: development/production
   - app.baseURL: Base URL aplikasi
   - app.forceGlobalSecureRequests: Force HTTPS
   - database.default.*: Database credentials
   - app.CSPEnabled: Content Security Policy

5. FITUR PWA:
   - service-worker.js di public/
   - manifest.json di public/
   - Bisa di-install sebagai app di mobile
   - Offline support (partial)

6. INTEGRASI WHATSAPP:
   - Via Notification::sendToWhatsApp() library
   - Trigger saat report submitted
   - Kirim ringkasan laporan ke nomor group/channel
